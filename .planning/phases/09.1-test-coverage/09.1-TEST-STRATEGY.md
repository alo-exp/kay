## 1. Testing pyramid

```
               /    UI automation (tauri-driver + WDIO)  \    Few, slow, highest confidence
              / ───────────────────────────────────────── \
             /          Compile-fail (trybuild)              \   Object-safety + macro expansion
            / ─────────────────────────────────────────────── \
           /               Property (proptest)                 \  Invariants over random inputs
          / ─────────────────────────────────────────────────── \
         /                  Integration (tests/ dir)            \  Module composition + pub API coverage
         / ──────────────────────────────────────────────────────\
        /                     Unit (inline + render)              \  Fastest; TUI compile + render
       /─────────────────────────────────────────────────────────────/
```

- **Unit (~25%)** — TUI compile test + render test via `ratatui::backend::TestBackend`; inline unit tests in `forge_*` crates
- **Property (~10%)** — `proptest` for `forge_api` JSON round-trips, `forge_domain` entity generation, `forge_embed` schema validation, `forge_stream` event buffering
- **Compile-fail (~3%)** — `trybuild` proc-macro expansion tests in `forge_tool_macros`
- **Integration (~50%)** — `tests/` directories per gap-list crate; `assert_cmd` for CLI-style crates; `insta` snapshots for display/render crates
- **UI automation (~10%)** — `tauri-driver` + WebDriverIO smoke suite for `kay-tauri` UI
- **Subprocess (~2%)** — Real `std::process::Command` escape tests for `kay-sandbox-{linux,macos,windows}` (on respective OS)

## 3. TDD Discipline

### RED → GREEN per wave

Every wave follows the TDD iron law:

1. **RED commit**: Write `tests/` directories + failing test stubs that reference non-existent assertions or expected behavior. Commit message prefix: `[RED] Wave N — failing tests for <feature>`
2. **GREEN commit**: Implement the passing tests. Commit message prefix: `[GREEN] Wave N — passing tests for <feature>`

No wave skips RED. No merging without GREEN.

### Commit message prefixes

```
[RED] Wave 1 — failing integration tests for forge_app, forge_config, ...
[GREEN] Wave 1 — passing integration tests for forge_app, forge_config, ...
[RED] Wave 4 — failing subprocess escape tests for kay-sandbox-*
[GREEN] Wave 4 — passing subprocess escape tests on 3 OSes
```

### Test organization per crate

Each gap-list crate gets a `tests/` directory:

```
crates/forge_app/tests/
  ping.rs         — assert_cmd; tests ping/handshake behavior
crates/forge_domain/tests/
  entity.rs       — proptest; entity generation + validation
crates/forge_tool_macros/tests/
  expand.rs       — trybuild; macro expansion fixtures
crates/kay-tauri/tests/
  commands.rs     — tauri::test::mock_builder() IPC tests
crates/kay-tui/tests/
  render.rs       — compile test + TestBackend render
```

## 5. CI strategy

### Matrix: ubuntu-latest + macos-14 + windows-latest

All three OSes run the full `cargo test --workspace` suite. OS-specific tests gate on `runs-on` via `matrix.os`.

| OS | Sandbox tests | Tauri UI tests | Notes |
|----|---------------|----------------|-------|
| ubuntu-latest | kay-sandbox-linux subprocess tests | WDIO smoke (secondary) | Primary CI |
| macos-14 | kay-sandbox-macos subprocess tests | WDIO smoke (primary) | GUI support enabled |
| windows-latest | kay-sandbox-windows subprocess tests | N/A (WDIO headless) | PowerShell compatible |

### Per-crate cargo test

```yaml
jobs:
  test:
    strategy:
      matrix:
        os: [ubuntu-latest, macos-14, windows-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - name: Install Rust
        uses: dtolnay/rust-action@stable
      - name: cargo test (crate group 1 — Wave 1/2 forge_*)
        run: cargo test -p forge_app -p forge_config -p forge_display -p forge_domain -p forge_embed -p forge_fs -p forge_infra -p forge_json_repair -p forge_main
      - name: cargo test (crate group 2 — Wave 2)
        run: cargo test -p forge_markdown_stream -p forge_repo -p forge_services -p forge_snaps -p forge_spinner -p forge_stream -p forge_template -p forge_tracker -p forge_walker
      - name: cargo test (crate group 3 — Wave 3)
        run: cargo test -p forge_api -p forge_ci -p forge_test_kit -p forge_tool_macros
      - name: cargo test (kay-sandbox-* — OS-gated)
        run: |
          if [ "${{ matrix.os }}" = "ubuntu-latest" ]; then
            cargo test -p kay-sandbox-linux
          elif [ "${{ matrix.os }}" = "macos-14" ]; then
            cargo test -p kay-sandbox-macos
          else
            cargo test -p kay-sandbox-windows
          fi
      - name: cargo test (kay-tauri)
        run: cargo test -p kay-tauri
      - name: cargo test (kay-tui)
        run: cargo test -p kay-tui
      - name: coverage gate
        run: sh scripts/coverage-gate.sh
      - name: WDIO smoke (macOS primary / Linux secondary)
        if: matrix.os == 'macos-14' || matrix.os == 'ubuntu-latest'
        run: cd crates/kay-tauri/ui && pnpm wdio run wdio.conf.ts
```

### Merge approval gate

All three OS CI jobs must be green for merge approval. No partial green.

## 6. Gap-closure verification

### coverage-gate.sh

`scripts/coverage-gate.sh` iterates the gap-list crates and fails CI if any crate has zero `tests/` directory or zero `#[test]` functions.

```bash
#!/usr/bin/env bash
set -e

GAP_CRATES=(
  "forge_app"
  "forge_config"
  "forge_display"
  "forge_domain"
  "forge_embed"
  "forge_fs"
  "forge_infra"
  "forge_json_repair"
  "forge_main"
  "forge_markdown_stream"
  "forge_repo"
  "forge_services"
  "forge_snaps"
  "forge_spinner"
  "forge_stream"
  "forge_template"
  "forge_tracker"
  "forge_walker"
  "forge_api"
  "forge_ci"
  "forge_test_kit"
  "forge_tool_macros"
  "kay-sandbox-linux"
  "kay-sandbox-macos"
  "kay-sandbox-windows"
  "kay-tauri"
  "kay-tui"
)

FAILED=0
for crate in "${GAP_CRATES[@]}"; do
  tests_dir="crates/$crate/tests"
  if [ ! -d "$tests_dir" ]; then
    echo "FAIL: $crate has no tests/ directory"
    FAILED=1
    continue
  fi

  test_count=$(grep -r "#\[test\]" "$tests_dir" --include="*.rs" 2>/dev/null | wc -l || echo 0)
  if [ "$test_count" -eq 0 ]; then
    echo "FAIL: $crate has tests/ dir but zero #[test] functions"
    FAILED=1
  else
    echo "PASS: $crate — $test_count test(s) found"
  fi
done

if [ $FAILED -ne 0 ]; then
  echo ""
  echo "ERROR: One or more gap-list crates failed coverage gate."
  echo "Post-merge push with a gap-list crate missing tests is a revert-worthy regression."
  exit 1
fi

echo ""
echo "All gap-list crates have test coverage."
exit 0
```

### CI integration

The script runs on every PR and post-merge push as the final CI step.
