<objective>
Close PROV-05 by adding the tolerant second-pass JSON parser (reusing ForgeCode's battle-tested `forge_json_repair::json_repair`) and rewiring the translator to use it.

Current state after plan 02-08: the translator uses STRICT serde_json only. A malformed tool_call argument emits `ProviderError::Stream` (or `ToolCallMalformed`) and TERMINATES the stream. This plan changes the behavior:

1. **Two-pass parser (D-03):** `src/tool_parser.rs` exposes `parse_tool_arguments(raw: &str) -> ParseOutcome` with three outcomes: `Clean(Value)` (pass-1 succeeded), `Repaired(Value)` (pass-1 failed, pass-2 `forge_json_repair` succeeded), `Malformed { error: String }` (both failed).

2. **Translator upgrade:** when the translator reaches a terminal marker and drains builders, it calls `parse_tool_arguments` instead of strict-only parse. Outcome:
   - `Clean(args)` / `Repaired(args)` → emit `AgentEvent::ToolCallComplete { id, name, arguments: args }` (the distinction between Clean and Repaired is observable via tracing-log, not via the AgentEvent variant — consumers shouldn't care).
   - `Malformed { error }` → emit `AgentEvent::ToolCallMalformed { id, raw: builder.arguments_raw, error }` (a DATA event, not a terminal error — the stream CONTINUES so subsequent tool_calls or text content can still deliver).

3. **1MB safety cap (TM-06):** every time a delta appends to `ToolCallBuilder::arguments_raw`, check if the new length exceeds 1,048,576 bytes. If yes, drop that call entirely: emit `AgentEvent::ToolCallMalformed { id, raw: "<truncated>".into(), error: "exceeds 1MB limit" }` and remove the builder from the map. Further deltas for that id are silently ignored (or re-added as a new builder; simplest is to ignore — the call is already lost).

4. **Never panic invariant proof — proptest:** `tests/tool_call_property.rs` runs two proptest invariants:
   - `parser_never_panics`: for any Unicode string, `parse_tool_arguments` must return without panicking.
   - `well_formed_json_always_clean`: for serde_json-serialized HashMap<String, i64>, parse_tool_arguments returns `ParseOutcome::Clean`.

5. **Integration test — malformed cassette:** `tests/tool_call_malformed.rs` loads `fixtures/sse/tool_call_malformed.jsonl` (the one with unquoted keys + trailing comma in the second delta) and asserts the stream DOES NOT emit ProviderError. It should emit either `ToolCallComplete` (if forge_json_repair successfully repaired the `{cmd: "ls,"}` form) OR `ToolCallMalformed` (if it could not). Either outcome is acceptable; what's NOT acceptable is a panic or an Error variant that terminates the stream. The test asserts no `AgentEvent::Error` variant is seen AND no panic.

Output: `src/tool_parser.rs` (new), refactored `src/translator.rs` (call parse_tool_arguments + enforce 1MB cap + upgrade Malformed emission path), 2 new integration tests, `lib.rs` module declaration.

use serde_json::Value;
use kay_core::forge_json_repair::json_repair;

#[derive(Debug)]
pub enum ParseOutcome {
    /// Pass-1 strict serde_json succeeded.
    Clean(Value),
    /// Pass-1 failed; pass-2 forge_json_repair succeeded.
    Repaired(Value),
    /// Both passes failed. `error` is a human diagnostic (from repair_err).
    Malformed { error: String },
}

pub fn parse_tool_arguments(raw: &str) -> ParseOutcome {
    if raw.is_empty() {
        return ParseOutcome::Clean(Value::Object(Default::default()));
    }
    match serde_json::from_str::<Value>(raw) {
        Ok(v) => ParseOutcome::Clean(v),
        Err(_strict_err) => {
            match json_repair::<Value>(raw.to_string()) {
                Ok(v) => ParseOutcome::Repaired(v),
                Err(repair_err) => ParseOutcome::Malformed { error: repair_err.to_string() },
            }
        }
    }
}

  //!
  //! Pass 1: serde_json::from_str (strict).
  //! Pass 2 (on pass-1 failure): kay_core::forge_json_repair::json_repair
  //!   (battle-tested against OpenRouter's real tool-call malformations —
  //!   trailing commas, unquoted keys, stringified numbers, markdown code-block
  //!   wrappers, etc.).
  //! Pass 3 (on pass-2 failure): ParseOutcome::Malformed with diagnostic.
  //!
  //! Invariant: NEVER PANIC. This is enforced by the crate-root lint
  //! `#![deny(clippy::unwrap_used, clippy::expect_used)]` and proven by
  //! the proptest in tests/tool_call_property.rs.

  use serde_json::Value;

  use kay_core::forge_json_repair::json_repair;

  /// Maximum bytes we accumulate into a single tool_call's `arguments_raw`
  /// buffer. Arguments larger than this are reported as Malformed rather
  /// than fed into the parser (TM-06).
  pub const MAX_TOOL_ARGS_BYTES: usize = 1_048_576;

  #[derive(Debug, Clone, PartialEq)]
  pub enum ParseOutcome {
      /// Pass-1 strict parse succeeded.
      Clean(Value),
      /// Pass-1 failed; pass-2 forge_json_repair succeeded.
      Repaired(Value),
      /// Both passes failed. `error` carries a human-readable diagnostic
      /// from forge_json_repair's Error display.
      Malformed { error: String },
  }

  /// Parse tool-call arguments raw bytes into a `serde_json::Value`.
  ///
  /// Empty input is treated as the empty object `{}` (OpenRouter variance
  /// per D-04: null / empty initial argument deltas are legal).
  pub fn parse_tool_arguments(raw: &str) -> ParseOutcome {
      if raw.is_empty() {
          return ParseOutcome::Clean(Value::Object(Default::default()));
      }
      // Pass 1: strict
      match serde_json::from_str::<Value>(raw) {
          Ok(v) => ParseOutcome::Clean(v),
          Err(_strict_err) => {
              // Pass 2: forge_json_repair (owned String).
              match json_repair::<Value>(raw.to_string()) {
                  Ok(v) => ParseOutcome::Repaired(v),
                  Err(repair_err) => ParseOutcome::Malformed {
                      error: repair_err.to_string(),
                  },
              }
          }
      }
  }

  #[cfg(test)]
  mod unit {
      use super::*;

      #[test]
      fn empty_input_is_empty_object() {
          let o = parse_tool_arguments("");
          assert_eq!(o, ParseOutcome::Clean(Value::Object(Default::default())));
      }

      #[test]
      fn well_formed_json_takes_strict_path() {
          let o = parse_tool_arguments(r#"{"cmd":"ls -la"}"#);
          match o {
              ParseOutcome::Clean(v) => {
                  assert_eq!(v["cmd"], Value::String("ls -la".into()));
              }
              other => panic!("expected Clean, got {other:?}"),
          }
      }

      #[test]

          // Canonical OpenRouter variance (PATTERNS lines 572-640).

          match o {
              ParseOutcome::Repaired(v) => {
                  assert_eq!(v["cmd"], Value::String("ls".into()));
              }
              other => panic!("expected Repaired, got {other:?}"),
          }
      }

      #[test]
      fn unquoted_keys_repaired() {
          let o = parse_tool_arguments(r#"{cmd: "ls"}"#);

      }
