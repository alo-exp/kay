# Phase 9.5 Plan ‚Äî TUI Frontend (ratatui)

**Goal:** Ship `crates/kay-tui/` as a production-quality ratatui terminal UI that consumes the `kay run --jsonl` JSONL event stream with live session view, tool-call timeline, and cost meter.

**Branch:** `phase/09.5-tui-frontend` (cut from `main` after Phase 9 merges)
**Requirements:** TUI-01..TUI-05
**Spec:** `docs/superpowers/specs/2026-04-24-phase9.5-tui-frontend-design.md` (Rev 1)

---

## Success Criteria

- `cargo check -p kay-tui` compiles clean on macOS + Linux + Windows
- `cargo test -p kay-tui` passes (all unit + integration tests)
- `kay-tui --help` renders the help text
- JSONL parser: `echo '{"type":"TextDelta","data":{"content":"hello"}}' | cargo run --quiet` renders "hello" in trace
- ratatui renders AgentTrace, ToolTimeline, StatusBar, PromptInput panes
- Keyboard navigation (Tab, arrows, Ctrl+C, q) works
- Subprocess lifecycle: spawn ‚Üí stream ‚Üí SIGTERM ‚Üí drain ‚Üí exit
- EventLog cap (10,000 events) with LRU eviction verified by test
- Color palette renders correctly in 256-color terminal
- SSH-safe: no mouse events, basic ANSI only

## Key Technical Decisions

- Ratatui 0.27+ with `crossterm` backend
- Subprocess: `tokio::process::Command::new("kay").args(["run", "--jsonl", ...])`
- JSONL: `BufRead::read_line` ‚Üí `serde_json::from_str` ‚Üí `TuiEvent`
- EventLog: 10,000 event cap (ring-buffer eviction)
- Error handling: `JsonlIterator` skips non-JSON lines with `tracing::debug!`
- Color: ANSI 256-color palette (16-color fallback)
- Key bindings: Tab (cycle pane), arrows (scroll), Ctrl+C (SIGTERM), q (quit)
